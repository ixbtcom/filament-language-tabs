# Проблема с EditorJS внутри Builder

## Симптом

Builder внутри которого EditorJS не переключается между локалями.

## Гипотеза

EditorJS - это Field компонент внутри Builder blocks. Текущая реализация обрабатывает только top-level компоненты, но НЕ обрабатывает вложенные поля.

Структура:
```
LanguageTabs::make([
    Builder::make('content')  ← Обрабатывается prepareBuilderForLocale ✅
        ->blocks([
            Block::make('text')
                ->schema([
                    EditorJS::make('body')  ← НЕ обрабатывается ❌
                ])
        ])
])
```

## Что происходит

1. LanguageTabs обрабатывает Builder через `prepareBuilderForLocale()`
2. Builder получает хуки для синхронизации с JSON
3. **НО**: EditorJS внутри Block НЕ получает хуки `prepareFieldForLocale()`
4. EditorJS не знает о механизме переводов
5. Данные не синхронизируются

## Почему это важно

Builder хранит данные в структуре:
```json
{
  "en": [
    {
      "type": "text",
      "data": {
        "body": {
          "blocks": [...],  ← EditorJS data
          "version": "2.0"
        }
      }
    }
  ]
}
```

Builder синхронизирует весь массив items, но **не знает** что внутри `data.body` должна быть специальная логика для EditorJS.

## Требуемое решение

Нужна **рекурсивная обработка** вложенных полей:

1. prepareBuilderForLocale обрабатывает Builder
2. **Дополнительно**: обрабатывает все поля внутри blocks
3. Каждое вложенное поле получает свои хуки
